Industry Solutions Process

How Industry Architects and the Field create, validate, and publish solutions.

Every Solution Has Two Required Components

M Solution Metadata

Industry, use case, personas, architecture, value prop. Created via ISF-Creator skill. Surfaces in catalogs and discovery.

+

P Solution Package (Plugin)

A CoCo plugin in a GitHub repo. Can take two forms: (1) a collection of skills wrapped as an interactive, contextualizable solution, or (2) a deployable pre-built demo with a validated talk track. Best practices TBD (Sho)

Open Questions

Plugin packaging: Sho to define structure, naming, testing requirements.
Solution packages required: Can we agree all solutions must have a solution package? If not, should we unpublish those without one?
ISF cleanup: Required before scaling. Establish baseline quality.
Metadata ↔ Package link: Technical mechanism to connect them.
Versioning: How do updates to existing solutions flow?
Internal quality bar: What are the exact criteria for a solution to pass Gate 1 (internal council review)?
External quality bar: What are the exact criteria for a solution to pass Gate 2 and be promoted to Snowflake Labs?
Review process & SLA: How will skill/plugin reviews be handled — who is assigned, what is the expected turnaround time, and what happens if a submission is rejected?
